# Master Context Document — WoW-Aligned Demo Enhancements

**Date:** 2026-07-19  
**Scope:** PRD 0002 implementation on top of the Wealth Morning Brief V1/V2 application

## Current architecture

The FastAPI route and market-data boundaries are unchanged: `GET /` fetches the market snapshot, sentiment, and headlines; `POST /generate` reuses the posted snapshot and headlines and never re-fetches yfinance. Jinja2 renders the page and HTMX replaces only `#brief-region`.

DeepSeek now returns one labelled plain-text response:

- `BRIEF` — two grounded narrative paragraphs
- `INVESTMENT_IDEAS` — two or three demonstration ideas
- `WATCH` — exactly three short catalysts
- `HOUSE_VIEW` — two or three simulated research observations

`llm/brief.py` parses those sections defensively. Missing optional blocks become empty lists; missing brief content or any API failure returns the existing safe fallback message. The same parsed brief is wrapped in a fixed client-email template, so no second LLM call is needed.

## Updated request flow

1. The RM may select a persona chip, which fills existing profile values and an editable sample portfolio mix without submitting.
2. The RM explicitly presses **Generate My Brief**.
3. `POST /generate` builds a profile only when tier, goal, and geography are present; a non-blank portfolio mix is included as client-provided, unverified context.
4. DeepSeek receives the posted snapshot, headlines, persona, and optional mix in one request.
5. The partial renders available sections only: brief, demo ideas, three-item watchlist, simulated research view, and fixed email draft.
6. Missing optional sections are hidden; the market snapshot remains usable if DeepSeek fails.

## UI and trust boundaries

- Generated investment ideas are labelled **demo**.
- The house-view panel is labelled **simulated** and **not official OCBC research**.
- A persistent footer states that the app is a demonstration using public data and AI-generated content, not official research, advice, or a bank recommendation.
- The documented V3 path is governed ingestion and retrieval from approved proprietary research with citations and audit logs; no unused ingestion stub or live bank corpus exists in this version.
- Brief and email copy controls share a synchronous clipboard fallback for permission-restricted browsers.

## Files with changed responsibilities

- `wealth-brief/llm/brief.py` — structured output contract, parser, fixed email builder, portfolio context, safe empty panels.
- `wealth-brief/main.py` — accepts optional portfolio mix and passes parsed sections into the partial.
- `wealth-brief/templates/index.html` — persona presets, portfolio input, persistent disclaimer.
- `wealth-brief/templates/partials/brief.html` — ideas, watch, simulated research, and email sections.
- `wealth-brief/static/app.js` — persona prefill, portfolio samples, robust brief/email copy.
- `wealth-brief/static/style.css` — new controls and content-panel presentation.

## Dependencies and configuration

No dependencies or environment variables were added. Existing `DEEPSEEK_API_KEY` and optional `FINNHUB_API_KEY` remain the only application keys. Static CSS and JavaScript query versions were incremented for deployment cache invalidation.
